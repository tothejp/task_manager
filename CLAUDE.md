# CLAUDE.md — 임무분담표(TaskShare) 개발 가이드

이 문서는 이 프로젝트에서 코드를 작성/수정할 때 따라야 할 작업 방식을 정의합니다. PRD(`task_manager_PRD.md`)가 기능의 "무엇"을 정의하고, 이 문서는 "어떻게 작업하는지"를 정의합니다.

## 0. 항상 먼저 할 일
- 새로운 기능을 구현하기 전에 `task_manager_PRD.md`의 관련 섹션을 먼저 확인한다.
- 작업을 마치면 `current_status.md`를 갱신한다 (진행 상황을 추적하는 단일 소스).
- PRD에 없는 기능을 임의로 추가하지 않는다. 추가가 필요하다고 판단되면 먼저 사용자에게 제안하고 PRD에 반영한 뒤 구현한다.

## 1. 기술 스택 (고정)
- Frontend: Next.js 14 (App Router), TypeScript, Tailwind CSS
- 상태 관리: Zustand (클라이언트 상태), React Query (서버 상태/캐싱)
- Drag & Drop: dnd-kit
- Backend/DB: Supabase (PostgreSQL + RLS + Auth)
- ORM: Prisma
- 배포: Vercel
- BookLog 프로젝트와 동일한 스택을 재사용하므로, 가능하면 BookLog에서 검증된 설정(ESLint, Prisma 스키마 패턴, Supabase 클라이언트 초기화 방식 등)을 그대로 따른다.

## 2. 핵심 설계 원칙

### 2.1 역할-플랫폼 분리는 타협 불가 원칙
- 팀원 = 모바일 전용, 관리자 = PC 전용(배정 작업) + 모바일(조회 전용)
- 화면/컴포넌트를 만들 때 항상 "이 화면은 누가, 어느 디바이스에서 보는가"를 먼저 명확히 하고 작성한다.
- 디바이스 감지 로직은 User-Agent 기반으로 1차 판단하고, 너무 공격적으로 차단하지 않는다 (안내 메시지 + 계속 진행 옵션 정도로 시작).

### 2.2 모든 "통제" 로직은 규칙 기반(rule-based)으로 구현
- 스킬 매칭, 시간 중복 판단, 자동배정 추천 정렬은 모두 명시적인 비교/정렬 로직으로 구현한다.
- LLM API를 호출하는 코드를 추가하지 않는다. PRD에 "AI API 불필요"로 명시되어 있다.
- 자동배정 알고리즘은 순수 함수로 분리해서 작성한다 (입력: 가용인원 목록 + 과업 정보 + 기존 배정 현황 → 출력: 추천 배정 결과). 테스트하기 쉬운 형태를 유지한다.

### 2.3 시간 겹침 판단은 공통 유틸로
- "동일 인원의 과업 시간이 겹치는지" 판단하는 로직은 여러 곳(D&D 드롭, 자동배정)에서 재사용되므로 공통 유틸 함수(예: `isTimeOverlapping(taskA, taskB)`)로 한 번만 구현하고 공유한다.

### 2.4 경고 모달은 차단이 아니라 확인
- 필수 스킬 미보유자 배정 시도 시: 차단하지 않고 모달로 확인 후 진행 허용. 이 정책을 구현할 때 "확인" 시 `assignments.skill_override = true`로 기록해서 나중에 추적 가능하게 한다.
- 시간 중복은 반대로 하드 차단이다. 둘을 혼동하지 않는다.

## 3. 데이터 모델 작업 시 주의사항
- PRD 6장의 데이터 모델 초안을 기준으로 Prisma 스키마를 작성한다.
- RLS(Row Level Security)는 Supabase에서 팀 단위로 데이터가 격리되도록 반드시 설정한다. 팀 A의 관리자가 팀 B의 데이터를 조회/수정할 수 없어야 한다.
- 스킬 태그(`skill_tags`)는 관리자만 쓰기 권한을 갖도록 RLS 정책을 분리한다 (팀원은 read-only).

## 4. 코드 작성 스타일
- 컴포넌트, 함수 등의 네이밍은 영어로, 주석과 사용자 노출 텍스트(에러 메시지, 모달 문구 등)는 한국어로 작성한다.
- 에러/경고 메시지는 PRD에 제시된 예시 문구의 톤을 따른다 (예: "OOO님은 '지게차운전' 스킬이 없습니다. 그래도 배정하시겠습니까?")
- 매직 넘버를 피하고 상수로 분리한다 (예: 공정성 지표의 ±20% 임계값).

## 5. 작업 순서 권장
1. 데이터 모델(Prisma 스키마) 확정 및 마이그레이션
2. 인증/팀 생성/구성원 초대 (3.1)
3. 팀원 모바일 일정 입력 화면 (3.2)
4. 관리자 PC 가용인원 대시보드 + 스킬 필터링 (3.3)
5. 과업 생성/관리 화면 (3.4)
6. D&D 배정 화면 (3.5) — 시간 중복 차단, 스킬 경고 모달 포함
7. 자동배정 로직 및 UI (3.6)
8. 휴가 등록 → 공백 처리 → 재배정 흐름 (3.7)
9. 완료 체크 (3.8)
10. 공정성 지표 시각화 (3.9)

이 순서는 의존성(예: 배정 기능은 가용인원/스킬 데이터가 먼저 있어야 함)을 고려한 권장 순서이며, 강제 규칙은 아니다.
