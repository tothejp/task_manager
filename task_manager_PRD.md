## 1. 개요

### 1.1 배경
소규모 팀(5~15명)에서 구성원별 업무(임무)를 배정하고 관리할 때, 휴가/부재로 인한 업무 공백과 특정 인원에게 업무가 몰리는 비효율이 자주 발생한다. 엑셀이나 종이 기반 분담표는 갱신과 공유가 번거롭고, 부재 상황을 실시간으로 반영하기 어렵다.

### 1.2 제품 한 줄 정의
팀원은 모바일로 본인의 일정·휴가·근무 가능 여부를 입력하고, 관리자는 PC에서 이를 종합해 가용 인원을 판단하여 과업을 배정하는 역할 분리형 임무 분담 관리 앱.

### 1.3 목표 사용자
- 소규모 팀(5~15명) 관리자 및 팀원
- 예: 사내 부서, 매장, 식당, 소규모 사무실 등 정기적으로 임무/당직을 분배하는 조직

### 1.4 핵심 가치
| 가치 | 설명 |
|---|---|
| 역할 분리 | 팀원=모바일 입력 전용, 관리자=PC 관리 전용으로 각자 최적화된 환경 제공 |
| 자동화 | 휴가/일정 등록 시 가용인원 자동 계산 및 재배정 후보 제안 |
| 공정성 | 누적 업무량 시각화로 편중 방지 |

## 2. 사용자 역할 및 플랫폼 정책

| 역할 | 플랫폼 | 권한 |
|---|---|---|
| 팀원 | **모바일 전용** | 본인 일정/휴가/근무/휴무일 입력, 본인 배정 확인 및 완료 체크 |
| 관리자 | **PC 전용(배정·관리)** + **모바일(조회 전용)** | PC: 가용인원 조회, 과업 생성·요구인원 설정, 배정·재배정, 통계 조회 / 모바일: 현황 조회만 가능(배정·생성 등 변경 작업 불가) |

> 플랫폼 강제는 UX 단순화를 위한 정책이며, 기술적으로는 반응형 웹으로 구현하되 역할과 디바이스에 따라 화면/기능을 제한한다. 관리자가 모바일로 접속하면 조회 전용 뷰(가용인원 현황, 배정 현황, 통계)만 노출되고, 배정·과업 생성 등 변경 작업은 PC 접속 시에만 활성화된다. 팀원이 PC로 접속 시 "모바일에서 이용해주세요" 안내.

### 2.1 In Scope (MVP)
1. 팀 생성 및 구성원 초대
2. **[팀원/모바일]** 개인 일정 입력: 근무 가능일, 휴가, 휴무일 등록
3. **[관리자/PC]** 특정 일자의 가용 인원 조회 (팀원 입력 데이터 종합)
4. **[관리자/PC]** 구성원별 스킬 부여/회수 (관리자만 가능, 본인 수정 불가)
5. **[관리자/PC]** 가용인원 중 특정 스킬 보유자 필터링 조회
6. **[관리자/PC]** 과업 생성 시 진행 시간(시작~종료), 요구 인원수, 필수 스킬 지정
7. **[관리자/PC]** Drag & Drop 방식의 과업-인원 배정, 필수 스킬 미보유자 배정 시 경고 모달로 1차 확인
8. **[관리자/PC]** 자동배정 기능 (스킬·가용성·업무량 기반 추천 결과를 일괄 배정)
9. 동일 인원의 시간대가 겹치는 과업 중복 배정 차단 (겹치지 않으면 같은 날 복수 과업 배정 허용)
10. 휴가/부재 등록 시 기존 배정 공백 표시 및 재배정 후보 제안
11. 임무 완료 체크 (팀원/모바일)
12. 구성원별 업무량 누적 통계 (공정성 지표, 관리자/PC)
13. 역할 권한: 관리자 / 팀원

### 2.2 Out of Scope (이후 단계)
- 외부 캘린더(구글/카카오) 연동
- 슬랙/카카오워크 알림 연동
- 멀티팀/조직 관리
- LLM 기반 자연어 추천/설명 (예: "왜 이 사람을 추천했는지 설명")
- 임무 템플릿 마켓플레이스

## 3. 핵심 기능 명세

### 3.1 팀 & 구성원 관리
- 관리자는 팀을 생성하고 초대 코드/링크로 구성원 초대
- 구성원 역할: `관리자`(PC, 배정/설정 권한), `팀원`(모바일, 일정 입력·완료체크·휴가등록)
- **가입 승인**: 이메일 인증 대신 관리자 승인 방식을 사용한다. 팀을 새로 만드는 관리자는 즉시 활성화되고, 초대 코드로 합류하는 팀원은 "승인 대기" 상태로 등록되어 관리자가 관리자 페이지에서 승인해야 정식 이용이 가능하다.
- 구성원 프로필: 이름, 역할, **보유 스킬 태그(다중 선택)**
- **스킬 태그는 관리자만 부여/회수 가능** (팀원 본인은 조회만 가능, 직접 추가/수정 불가). 자격증, 면허, 교육 이수 등 객관적 기준에 따라 관리자가 부여하는 방식을 전제로 한다.

### 3.2 [팀원/모바일] 개인 일정 입력
- 입력 항목: 휴가(기간), 휴무일(반복 가능, 예: 매주 일요일), 근무 가능 시간대(선택)
- 모바일 화면에서 캘린더 UI로 날짜를 선택해 상태 등록 (휴가/휴무/근무가능)
- 등록 즉시 관리자 화면의 가용인원 데이터에 반영된다 (관리자 승인 절차 없음)

### 3.3 [관리자/PC] 가용인원 판단
- 날짜를 선택하면 해당 일자의 전체 구성원 목록을 "가용 / 휴가 / 휴무"로 자동 분류해 표시
- 가용 인원 수, 명단을 한눈에 확인 가능한 대시보드 뷰
- **스킬 필터링**: 특정 스킬 태그를 선택하면 가용 인원 중 해당 스킬 보유자만 별도로 강조/필터링하여 표시 (예: "이 날 용접 가능 인원 3명 중 가용 2명")
- 스킬별 보유 인원 수와 그중 가용 인원 수를 요약 카드로 함께 제공

### 3.4 [관리자/PC] 과업(Task) 관리
- 과업 속성: 이름, 설명, 날짜, **진행 시간(시작 시각~종료 시각)**, **요구 인원수**, **필수 스킬 태그(선택, 1개 이상 지정 가능)**, 반복 주기(없음/매일/매주/매월)
- 필수 스킬이 지정된 과업에 해당 스킬을 보유하지 않은 구성원을 배정하려 하면 경고 모달을 띄워 1차 확인을 거치도록 한다 (강제 차단은 아니며, 관리자가 사유를 인지한 상태에서 진행 여부를 직접 결정)
- 과업 생성 시 해당 일자 가용인원 중 요구 인원수만큼 배정 필요

### 3.5 배정(Assignment): Drag & Drop

- 화면 구성: 좌측에 해당 일자 가용 인원 목록(스킬 태그, 해당일 기배정 시간대 표시), 우측에 과업별 슬롯(요구 인원수만큼 빈 칸, 진행 시간 표시)
- 관리자는 가용 인원 카드를 드래그하여 과업 슬롯에 드롭하는 방식으로 배정
- **필수 스킬 경고**: 과업에 필수 스킬이 지정된 경우, 해당 스킬이 없는 인원을 드롭하면 경고 모달이 표시된다 ("OOO님은 '지게차운전' 스킬이 없습니다. 그래도 배정하시겠습니까?"). 관리자가 모달에서 확인을 누르면 배정이 진행되고, 취소를 누르면 배정이 취소된다. 즉, 시스템은 강제 차단하지 않고 1차 확인 절차만 거친다.
- **시간 중복 차단**: 동일 인원이 같은 날 이미 배정된 다른 과업과 **진행 시간이 겹치는 경우** 드롭이 차단된다 ("OOO님은 14:00~16:00에 이미 'B과업'에 배정되어 있어 배정할 수 없습니다"). 진행 시간이 겹치지 않으면 같은 날 여러 과업에 중복 배정이 허용된다.
- 요구 인원수를 초과하여 드롭 시 추가 배정 거부, 미달 시에는 슬롯이 빈 채로 남아 "미충원" 상태로 표시

### 3.6 자동배정 기능

- 관리자가 "자동배정" 버튼을 누르면, 해당 일자(또는 기간)의 미배정 과업 슬롯에 대해 시스템이 일괄로 인원을 추천 배정
- **자동배정 로직 (규칙 기반):**
  1. 해당 일자 가용 인원 중, 배정 대상 과업과 **시간이 겹치는 다른 과업에 이미 배정된 인원은 후보에서 제외**
  2. 남은 후보 중 과업의 필수 스킬 보유자를 우선 후보로 구분 (미보유자는 차순위로 분리하되 완전히 배제하지는 않음 — 자동배정은 보수적으로 스킬 보유자만 우선 채우고, 보유자가 부족하면 미충원으로 남기고 관리자가 수동으로 경고 모달을 거쳐 결정하게 한다)
  3. 후보 중 누적 배정 업무량이 적은 사람부터 우선 배정
  4. 동률 시 최근 배정일이 오래된 사람 우선
  5. 스킬 보유 후보가 요구 인원수보다 적으면 가능한 만큼만 배정하고 나머지는 "미충원"으로 표시, 사유 안내 (예: "필요 인원 3명 중 2명만 배정됨 — 해당 스킬 보유 가용 인원 부족")
- **시뮬레이션(미리보기)**: 자동배정 결과는 즉시 확정되지 않고 먼저 미리보기로 제시되며, 관리자가 결과를 확인하고 개별 항목을 D&D로 수정한 뒤 "확정" 버튼으로 일괄 반영
- 자동배정은 미배정 슬롯에 대해서만 동작하며, 이미 수동 배정된 슬롯은 덮어쓰지 않음

> **AI API 필요 여부에 대한 답변:** 스킬 매칭, 시간 중복 판단, 추천 정렬은 모두 태그/시간 비교와 업무량 비교라는 규칙 기반 로직으로 구현 가능하며 LLM API가 필요하지 않다. 자연어로 추천 이유를 설명하거나, 텍스트 기반 이력서/메모에서 스킬을 자동 추출하는 등의 고급 기능을 추가하려면 LLM API가 필요해지므로, 해당 기능은 Phase 2 이후로 분리한다.

### 3.7 휴가/부재 반영 및 재배정
- 이미 배정된 과업 기간에 휴가가 새로 등록되면 해당 배정은 자동으로 "공백" 상태로 전환
- 공백 발생 시 3.6과 동일한 자동배정 로직으로 재배정 후보를 제시 (관리자가 미리보기 확인 후 확정)

### 3.8 완료 체크 & 이력 (팀원/모바일)
- 팀원이 본인 배정 과업에 완료 체크
- 미완료 과업은 관리자 대시보드(PC)에 강조 표시
- 월별/기간별 완료율 조회 (관리자/PC)

### 3.9 공정성 지표 (관리자/PC)
- 구성원별 누적 배정 횟수를 막대그래프로 시각화
- 평균 대비 ±20% 이상 편차 시 경고 표시 (예: "OOO 업무 과부하")

## 4. 비기능 요구사항
- 역할별 접속 디바이스 안내: 관리자 PC 전용, 팀원 모바일 전용 (반응형 웹으로 구현, 접속 디바이스 감지 후 안내)
- 데이터는 팀 단위로 격리 (RLS 적용)
- 동시 편집 시 마지막 저장 우선 (충돌 알림은 이후 단계)

## 5. 기술 스택 (BookLog와 동일 스택 재사용)
- Frontend: Next.js 14, TypeScript, Tailwind CSS, Zustand, React Query
- Drag & Drop: dnd-kit (React 기반, 접근성·성능 고려)
- Backend/DB: Supabase (PostgreSQL, RLS, Auth)
- ORM: Prisma
- 배포: Vercel
- 인증: Supabase Auth (이메일 또는 소셜 로그인)

> 비고: 스킬 매칭/통제/자동배정은 모두 규칙 기반 로직으로 구현하므로 LLM API, 벡터 검색(pgvector) 등은 MVP에서 불필요. 구성원 스킬 태그와 과업 필수 스킬 태그는 단순 배열 컬럼 또는 조인 테이블로 관리.

## 7. 향후 로드맵 (Phase 2+)
1. 외부 캘린더/메신저 알림 연동 및 배정 변경 푸시 알림 (카카오톡/슬랙 등)
2. 스킬 보유 여부(Y/N)를 넘어선 숙련도 등급(상/중/하) 도입 및 추천 정렬 고도화에 활용

4. 임무 템플릿 저장 및 재사용

## 8. 정책 확정 사항
- **휴가 등록**: 관리자 승인 절차 없이 즉시 반영
- **관리자 모바일 접속**: 조회 전용으로 허용 (배정/생성 등 변경 작업은 PC에서만)
- **수익 모델**: 유료화 계획 없음, 무료 운영 전제로 설계 (B2B 플랜 분기 불필요)
- **필수 스킬 미보유자 배정**: 강제 차단이 아닌 경고 모달을 통한 1차 확인 후 허용
- **중복 배정**: 동일 인원의 과업 진행 시간이 겹치는 경우에만 차단, 시간이 겹치지 않으면 같은 날 복수 과업 배정 허용
- **배정 변경 알림 / 스킬 숙련도 등급**: MVP 범위에서 제외, Phase 2에서 검토
- **회원 가입 인증 방식**: 이메일 인증 대신 관리자 승인 방식 채택. 초대 코드로 합류한 팀원은 관리자가 승인하기 전까지 "승인 대기" 상태로 팀 데이터에 접근할 수 없다 (팀을 새로 만드는 관리자는 예외로 즉시 활성화)
